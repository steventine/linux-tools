# Photo Timeline Viewer

A local photo browser with an OneDrive-style justified gallery layout. Photos and videos are displayed newest-first in dense justified rows, grouped by day and month, with a jump-to-month picker.

## Prerequisites

### Required: exiftool (highly recommended)

Without exiftool, dates fall back to file modification times (not the actual date the photo was taken).

```bash
sudo apt install libimage-exiftool-perl
```

### Optional: ffmpeg (for video thumbnails)

Without ffmpeg, videos will show a ▶ placeholder instead of a thumbnail frame.

```bash
sudo apt install ffmpeg
```

### Required: Node.js + npm

```bash
# Check if installed
node --version   # needs v18+
npm --version
```

## Installation

```bash
cd photoTimelineViewer
npm install
```

## Configuration

Edit `config.json` to point at your photo directories:

```json
{
  "port": 3001,
  "scans": [
    {
      "name": "My Photos",
      "dirs": [
        "/mnt/c/Users/YourName/Pictures"
      ]
    }
  ]
}
```

- **`port`**: Starting port (auto-increments if in use)
- **`scans`**: Named scan profiles, each with one or more directories
- **`dirs`**: Array of absolute paths to scan recursively

Multiple directories in one scan are combined and sorted together by date.

## Usage

```bash
npm start
```

The server will print the URL, e.g.:

```
  Photo Timeline Viewer → http://localhost:3001
```

Open that URL in your browser.

### First scan

1. Select a scan from the dropdown
2. Click **↻ Scan** — this runs exiftool across all configured directories (may take several minutes for large libraries)
3. Results are cached to `cache_<name>.json` so subsequent loads are instant

### Navigating

- **Scroll** up/down to move through time (newest at top)
- **Jump to Month** button opens a month picker — click any month to jump there
- **Click a photo** to open the full-size lightbox
- In the lightbox: **arrow keys** or on-screen arrows to navigate, **Esc** to close

## Architecture

```
exiftool (one batch pass)
    │
    └──► cache_<name>.json  ──►  server.mjs  ──►  browser
                                      │
                              .thumbnails/   (sharp-generated JPEG cache)
```

- **Thumbnails** are generated by `sharp` on first request and cached in `.thumbnails/`. They are max 800×800px maintaining aspect ratio. For videos, `ffmpeg` extracts a frame at the 1-second mark.
- **Justified layout** is computed in the browser from each photo's stored aspect ratio (from EXIF `ImageWidth`/`ImageHeight`). Falls back to 4:3 for images and 16:9 for videos when dimensions are unavailable.
- **Dates** use EXIF `DateTimeOriginal` → `CreateDate` → file mtime, in that priority order.

## HEIC / iPhone photos

`sharp` supports HEIC files if your system has libheif. If thumbnails fail for `.heic` files:

```bash
sudo apt install libheif-dev
npm rebuild sharp
```

## Supported formats

**Images:** jpg, jpeg, png, gif, webp, bmp, tiff, heic, heif, avif  
**Videos:** mp4, mov, avi, mkv, wmv, m4v, 3gp, mts, m2ts
